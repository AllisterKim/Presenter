# Presenter — 화면 발표 보조 플로팅 앱 PRD (v0.2)

| 항목 | 내용 |
|---|---|
| 문서 버전 | v0.2 (Open Questions 반영) |
| 작성 목적 | 개발 착수를 위한 요구사항 정의 및 세부 기능 구체화 |
| 대상 OS | Windows 11 Enterprise 24H2 |

## 1. 배경 및 문제 정의

WebEx, Teams 등으로 원격 회의를 하거나 빔프로젝터로 PC 화면을 공유해 발표할 때, 발표자는 다음과 같은 어려움을 겪습니다.

- 마우스 커서만으로는 화면상의 특정 지점을 강조하기 어려움 (레이저 포인터 부재)
- 설명 중 화면에 직접 도형/선/글자를 표시하고 싶어도 별도 프로그램을 켜야 함
- 발표 중 특정 영역만 캡처하거나, 스크롤이 필요한 긴 문서/웹페이지를 통째로 캡처하기 어려움
- 발표 시간 관리(타이머)를 위해 별도 앱을 띄워야 해서 화면 전환이 번거로움

**Presenter**는 이 모든 기능을 하나의 상시 플로팅 마스코트(캐릭터) 안에 통합하여, 발표자가 화면 공유를 유지한 채로 클릭 몇 번만으로 포인팅·주석·캡처·시간관리를 할 수 있게 하는 것을 목표로 합니다.

---

## 2. 목표 (Goals)

1. 화면 공유 중에도 항상 최상단에 보이는 캐릭터형 플로팅 UI 제공
2. 클릭 한 번으로 프레젠테이션/타이머/그리기 모드에 진입
3. 화면 위에 실시간으로 레이저포인터, 도형, 자유선, 텍스트 주석 표시
4. 영역 캡처 및 스크롤(긴 화면) 캡처 지원
5. Windows 11 Enterprise 환경에 최적화된 설치/실행 경험 제공 (설치형/포터블, 부팅 시 자동 실행)

## 3. MVP 범위 및 단계별 로드맵 (신규 — 24시간 목표 반영)

시간 제약(24시간)을 고려해, 전체 기능을 3단계로 나눕니다. **아래 우선순위는 리뷰 후 조정 가능합니다.**

### 3.1 Day-1 MVP (24시간 목표 — 반드시 포함)

| # | 기능 | 최소 구현 범위 |
|---|---|---|
| 1 | 플로팅 마스코트 | 고정 이미지 1종(자체 창작 캐릭터), Always-on-top, 드래그 이동만 지원 (좌/우 반전 애니메이션은 생략 가능) |
| 2 | 클릭 메뉴 | 좌클릭 시 3개 버튼(프레젠테이션/타이머/그리기)이 뜨는 단순 팝업 (원형 애니메이션 메뉴는 생략, 사각 버튼 리스트로 충분) |
| 3 | 레이저 포인터 | 고정 색상(빨강) 원형 포인터, 마우스 추적만 구현 (색상 커스터마이징/트레일 효과는 제외) |
| 4 | 간단 그리기 | 자유곡선(Freehand) 1종 도구만, 고정 두께 3px, 색상 5종 프리셋 버튼 선택 (RGB 피커/도형/텍스트는 제외) |
| 5 | 실행 방식 | 단일 EXE 실행 (설치 마법사 없이 바로 실행되는 형태), 관리자 권한 불필요 |
| 6 | 종료 | 트레이 아이콘 우클릭 → 종료 |

> Day-1 MVP는 "동작 검증 + 데모 가능" 수준입니다. 타이머, 캡처(영역/스크롤), 도형/텍스트 도구, RGB 컬러피커, 설치 마법사(Inno Setup), 자동시작 등록, 캐릭터 방향 애니메이션은 **모두 이후 버전으로 이연**합니다.

### 3.2 v1.1 (1차 정식 배포, 목표: MVP 이후 1~2주)

- 그리기 모드 전체 도구 세트 (직선/사각형/원·타원, RGB 컬러피커, 두께 조절, Undo/Redo, 텍스트 도구)
- 타이머 모드 (카운트다운/스톱워치)
- 영역 캡처 (클립보드 복사 + 파일 저장, 저장 경로 1회 지정 후 재사용)
- Inno Setup 기반 설치 파일(EXE) + "Windows 시작 시 자동 실행" 옵션
- 마스코트 이동 방향 애니메이션, 클릭 통과(Click-through) 토글

### 3.3 v1.2 (안정화/부가 기능)

- 스크롤 캡처 (Auto-scroll & Stitch)
- 캡처 후 즉시 주석 편집 연계
- 멀티 모니터 DPI 대응 검증, 화면공유 캡처 호환성 검증(WebEx/Teams/Zoom)
- 다국어(영문 UI), 접근성 옵션

## 5. 사용자 시나리오 (User Flow)

1. 발표자가 Presenter.exe를 실행하면 화면 한쪽에 캐릭터가 플로팅으로 나타난다.
2. 발표자가 WebEx/Teams에서 화면 공유를 시작한다 (마스코트도 공유 화면에 함께 노출됨).
3. 특정 위치를 강조하고 싶을 때 → 마스코트 클릭 → **프레젠테이션 모드** → 레이저 포인터가 마우스를 따라다님.
4. 강조 표시를 남기고 싶을 때 → **그리기 모드** → 원하는 색상 선택 후 화면 위에 직접 드로잉 (v1.1부터 직선/도형/텍스트 가능).
5. (v1.1+) 화면을 캡처해서 자료로 남기고 싶을 때 → 캡처 도구 → 영역 지정 캡처 → 클립보드 복사/저장.
6. (v1.1+) 발표 시간 관리 → **타이머 모드** → 카운트다운 오버레이 표시.
7. 발표가 끝나면 트레이 아이콘에서 종료.

## 6. 세부 기능 요구사항

### 6.1 플랫폼 / 배포

| 구분 | 요구사항 |
|---|---|
| 대상 OS | Windows 11 Enterprise 24H2 (x64) 최적화 |
| 배포 방식 | 개별 사용자 직접 다운로드. **MSI 불필요** — 단순 EXE(포터블) + (v1.1부터) Inno Setup 기반 설치 EXE |
| 코드 서명 | **인증서 미보유** → 배포 파일은 미서명 상태. Windows SmartScreen "알 수 없는 게시자" 경고가 뜰 것으로 예상됨. 다운로드 페이지/안내문에 "추가 정보 → 실행" 우회 방법을 안내 문구로 반드시 포함 |
| 자동 시작 | v1.1부터: 설치 시 "Windows 시작 시 자동 실행" 체크박스 제공 (레지스트리 Run 키 등록). Day-1 MVP는 수동 실행만 지원 |
| 권한 | 관리자 권한 불필요. 전역 오버레이/마우스 후킹이 보안 정책(그룹정책)에서 차단될 가능성은 리스크로 기록 (8절) |
| 업데이트 | 자동 업데이트 없음 (수동 재배포/재다운로드) |
| 멀티 모니터 | v1.2에서 정식 검증 (Day-1/v1.1은 주 모니터 기준 동작을 우선 보장) |

| 기능 | 세부 내용 |

| 표시 방식 | Always-on-top 투명 배경 윈도우, 화면 어디로든 자유 배치 |

## 7. 비기능 요구사항

| 구분 | 요구사항 |
|---|---|
| 성능 | 오버레이 렌더링으로 인한 CPU/GPU 부하 최소화, 드로잉 시 프레임 드랍 없어야 함 |
| 리소스 | 유휴 상태 메모리 사용량 목표 100MB 이하 |
| 안정성 | 화상회의 앱(WebEx/Teams/Zoom)과 동시 실행 시 충돌 없어야 함 |
| 보안 경고 대응 | 미서명 배포이므로 SmartScreen/백신 경고 발생 가능 → 사용자 안내 문서(FAQ) 필수 제공 |
| 저장 데이터 | 모든 캡처/설정은 로컬 저장만 수행, 외부 전송 없음 |
| 다국어 | 1차: 한국어만 (영문은 v1.2 검토) |
| 로깅 | 오류 발생 시 로컬 로그 파일 생성 |

## 8. 기술 스택 후보 (검토용)

| 옵션 | 장점 | 단점 |
|---|---|---|
| C# / .NET (WPF or WinUI 3) | Windows 네이티브, 투명/오버레이 창 구현 용이, 빠른 EXE 패키징 | - |
| Electron + Native 모듈 | 빠른 UI 개발 | 전역 오버레이/후킹은 Native 모듈 필요, 리소스 사용량 높음, 24시간 내 세팅에 불리 |

> **제안:** 24시간 내 Day-1 MVP 완성을 위해 **C# WPF** (Transparent Layered Window + 전역 마우스 후킹 P/Invoke) 조합을 강력 권장합니다. 러닝커브가 낮고 오버레이/투명창 예제 리소스가 풍부해 초기 세팅 속도가 가장 빠릅니다.

## 9. 리스크 및 제약사항

- **일정 리스크 (최우선)**: 24시간은 "마스코트+포인터+간단 그리기"만으로도 빠듯한 일정입니다. 화면공유 중 오버레이 정상 노출 검증, 전역 마우스 후킹 안정화에 예상보다 시간이 소요될 수 있어 3.1절 범위를 넘지 않도록 엄격히 통제해야 합니다.
- **SmartScreen/백신 경고**: 미서명 배포이므로 최초 실행 시 사용자가 경고를 무시하고 실행해야 함 → 첫 실행 가이드(스크린샷 포함 FAQ) 필요.
- **화면 공유 캡처 누락 리스크**: (완화됨) Day-1 검증 범위는 오버레이가 **로컬 화면 위에서 정상 표시**되는지까지이며, WebEx/Teams 등 화상회의 프로그램의 화면 공유 캡처에 오버레이가 함께 잡히는지는 Day-1 필수 검증 대상이 아님.
- **전역 후킹 오탐**: 백신/EDR이 키로거로 오인할 가능성 → 리스크로 기록, 서명 예산 확보 시 재검토.
